Objetivos do projeto

Atualizado 18/06/2020

Site para divulgação do projeto

Um dos objetivos do Projeto tinha como objetivo criar um site para a realização da divulgação do projeto, que é este que vocês estão vendo agora, nós primeiros fizemos um esboço de como seria o site como se pode ver na imagem, então fizemos toda a parte HTML, CSS e JS, então depois aplicamos Bootstrap para o site ficar responsívo e tivemos que refazer muita coisa do CSS, primeiramente começamos a fazer a página home do site e após isso fizemos a página de contato, que para fazer funcionar sem o uso de PHP usamos um site chamado Formspree que nos possibilitou enviar as mensagens diretamente para o email de um dos integrantes do grupo.

Após fazermos os outros objetivos das matérias fizemos esta página dos objetivos que busca mostrar todos os objetivos que fizemos duranto o semestre, então nós fizemos a página de mídia com os vídeos e imagens feitos sobre o projeto

Hospedagem do site

O outro objetivo do projeto referente ao anterior, era hospedar nosso site de forma gratuita, então pesquisamos muito a respeito disso até encontrarmos o awardspace que além de hospedar o site de forma gratuita ainda nos dava um subdominio de forma gratuita, então isso foi muito bom.

Além disso o awardspace é um site muito fácil de usar, basta criar a conta e escolher um dominio e então fazer o upload dos arquivos do seu site que ele já vai ficar online.

Sistema de entradas

Um dos objetivos do nosso projeto era fazer um sistema referente ao nosso projeto então fizemos um sistema que cadastrava entradas de uso do sistema do Arduino e podia consultalas e editalas, assim como excluilas, fazendo você ter um melhor controle de uso do Arduino para saber quando gastava mais e menos com água.

Primeiramente foi desenvolvido o programa em console usando o C# para termos uma melhor nossão de como fazer e nossas limitações, então em seguida foi feito sua versão MVC que é totalmente responsíva devido ao Bootstrap, então pode acessar o sistema de qualquer dispositivo. Para armazenar os dados das entradas no começo era usado um bloco de notas, mas em seguida foi migrado para um banco de dados chamado LiteDB, que é um banco de dados muito simples de ser usado.

Windows Server 2016

Alguns outros objetivos do projeto era instalar um windows server, instalar o site no windows server e gerenciar o AD para 10 host e 10 usuários.

A instalação do Windows Server 2016 foi bem de boa, já que a instalação não é complicada, mas após isso nós fizemos algumas configuração básicas como alterar o nome do servidor, configurar seu IP e baixar as instações do Windows Update, após isso nós baixamos o AD que também não foi algo complicado e criamos uma Unidade Organizacional onde adicionamos os hosts e usuários, e depois disso configuramos as permissões desses usuários, então instalamos o IIS no servidor e colocamos o site dele rodando no localhost.

Três Pilares da SI

Outros objetivos envolvendo o servidor era aplicar os 3 pilares da SI neles e para isso escolhemos o AD na confidencialidade, o Cobian na Integridade e o Nobreak na disponibilidade.

O AD como já falamos foi aplicado no servidor para dar somente as permissões nessesárias as pessoas melhorando assim a segurança e confidencialidade, o Cobian foi um progrma usado para fazer backups dos dados do servidor, então caso os dados forem perdidos teria um cópia deles salva, o Cobian permite programar backups dos dados então não é necessário ficar executando o programa toda vez o que é muito bom, já o Nobreak é um equipamento parecido com um estabilizador de além de manter a corrente estável ainda possui baterias que permitem que o servidor fique executando mesmo em falta de energia.
